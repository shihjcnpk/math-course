七八年级数学系统复习课程使用说明

知识脉络为主，教材章节为边界。按“进入状态—识别题型—规范作答—错题闭环—间隔复习”完成每一次学习。

课程定位

这不是按教材目录机械重排的网课，而是一套跨七、八年级的知识网络式复习系统。

数与式 方程与不等式 函数 几何推理 数据统计
学生

按短时单元完成课程，先识别主线和题型，再下笔。

家长

一次只盯一个动作，陪伴不超过20分钟。

教材同步

通过“教材章节”按七上、七下、八上、八下查课。

系统复习

通过首页8个模块按知识脉络复习。

1. 如何启动课程

推荐：一键启动

  1. 打开 D:\tools\math-course。
  2. 双击 start-course.bat。
  3. 首次使用等待依赖安装完成。
  4. 浏览器会打开 http://127.0.0.1:5173。
  5. 学习结束后，关闭课程命令窗口即可停止服务。
运行条件：电脑需要安装 Node.js 和 npm。若提示“npm was not found”，请先安装 Node.js。

手动启动

cd D:\tools\math-course
npm install
npm run dev -- --host 127.0.0.1

2. 第一次使用从哪里开始

跟随学校进度

点击顶部“教材章节”，按第1—24章查找对应复习课。

基础薄弱或系统复习

按首页“总地图→数与式→方程→几何→函数→统计→综合→复盘”学习。

知道薄弱知识点

用“搜索”查知识点、题型或易错点，再查看前置课程。

不知道从哪补

打开“知识图谱”或“错题本”，从最基础的确认断点开始。

注意：跨年级课程顺序是复习顺序，不是学校教材首次授课顺序。

3. 每节课的正确学习流程

确认知识网络位置先看所属主线、教材章节、前置知识、核心问题、后续关联和易混点。前置知识不懂时先回补。
完成3分钟启动只完成“今天一件事”和热身题，做完后再看答案。
学习概念与方法理解知识从哪里来、解决什么问题，不只背公式。
执行聚焦提示出现“暂停一下、检查一下、想一想、说一说”时，先完成提示再继续。
学习题型网络说出核心考点、第一眼、第一步、易混点、变式和常见错因。
独立完成变式至少做一道变式题，不能照抄例题过程。
记录错误并复习保留错误过程，选择错因，进入D0—D14复习。
勾选完成标准只有能独立作答、说明错因和前后关系，才标记“已掌握”。

短时学习节奏

单元时间要求
课前启动3分钟热身并进入状态
概念讲解8分钟说清概念来源和用途
例题讲解10分钟识别题型和第一步
休息3分钟离开屏幕、活动、喝水
变式训练10分钟独立作答
错题复盘5分钟写错因和提醒语
总结2分钟说出主线、核心和第一步

4. 做题前5秒刹车

  1. 圈关键词和限制词；
  2. 写清题目要求什么；
  3. 判断题型并说出第一步；
  4. 回忆所属知识主线；
  5. 确认后再下笔。
重点不是数满5秒，而是建立“先停、再想、再做”的固定动作。

5. 错题本如何使用

做错题时必须保留原错误过程，再填写主要错因、必要的次要错因和一句具体原因。

审题错误概念错误方法错误 计算错误步骤错误检查错误迁移错误

订正后的错题四问

  1. 我错在哪里？
  2. 我为什么会这样想？
  3. 下次看到类似题，第一步做什么？
  4. 换一道同类型题，能不能独立做对？

自动诊断

进入顶部“错题本”，点击“全部自动诊断”。有真实错误证据的节点才作为确认断点；只有前置关系的节点是自测候选。

6. D0—D14间隔复习

阶段时间任务
D0当天不看答案重做原题并写错因
D1第二天做同类型基础题
D3第三天做条件或问法变化的变式题
D7一周后混入其他题型，先识别再解答
D14两周后完成包含该知识点的综合题
真正掌握：D14能够独立做对、步骤完整，并能解释第一步为什么这样做。

7. 知识图谱怎么用

  1. 找到当前不会的知识点；
  2. 查看它的前置节点；
  3. 判断前置知识是否也薄弱；
  4. 先修复最基础的薄弱节点；
  5. 沿图谱回到原问题。

每次最多追溯一到两层，不要一次打开整张图中的所有节点。

8. 学习进度怎么理解

未开始

尚未进入课程。

学习中

正在完成短时单元。

需要复习

基本理解但还不稳定。

容易出错

同类错误反复出现。

已掌握

能独立完成变式和延迟复习。

总体百分比按“已掌握”计算，不按打开过多少课程计算。

9. 家长如何陪伴

每次陪伴不超过20分钟，一次只观察一个具体习惯。

不要说：“这么简单你怎么又错？”
可以说：“这次不是新问题，我们今天只处理这一类旧错误。”
  1. 提醒孩子完成5秒刹车；
  2. 让孩子自己说出错因和第一步；
  3. 孩子自己查出错误时，先肯定自查行为。

不要直接讲完整答案，也不要一次纠正多个习惯。

10. 推荐的一周安排

时间学习任务
周一新课1节 + D0
周二D1快测 + 新课1节
周三只处理错题和前置断点
周四周一内容D3变式 + 新课1节
周五混合练习或口头讲题
周末本周总结，不集中补大量新课
下周一完成周一内容D7混合
两周后完成D14综合检查

基础不稳时，每周学习3节新课即可，不要求快速刷完48讲。

11. 学习记录保存说明

  • 课程状态、错题、复习记录和完成标准保存在当前浏览器的 localStorage。
  • 当前没有账号和云同步。
  • 更换浏览器、使用隐私模式或清理网站数据可能导致记录丢失。
  • 建议固定使用同一台电脑和同一个浏览器配置文件。

13. 常见问题

双击脚本后没有打开页面
等待命令窗口出现“Local”地址，再手动打开 http://127.0.0.1:5173。
5173端口被占用
执行 npm run dev -- --host 127.0.0.1 --port 5174，再打开 http://127.0.0.1:5174。
手机上公式太长
公式区域支持横向滚动，直接左右滑动公式区域，不要缩小整个页面。
没有看到动画
并非每节课都有动画。先确认当前课程是否包含“动图演示”模块。
错题记录消失
检查是否更换浏览器、使用隐私模式或清理了网站数据。当前版本不能从云端恢复。

最重要的使用原则

知道本节在知识网络中的位置 → 看清题目 → 判断题型 → 说出第一步 → 独立完成 → 分类错因 → 做对变式 → 完成间隔复习。

不要把课程当成需要快速看完的48篇文章。真正目标是逐步减少同类错误，并能把旧知识迁移到新问题。
