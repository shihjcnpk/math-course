# 七八年级数学系统复习课程使用说明

> 适用对象：七、八年级学生、家长及课程维护者  
> 教材范围：人教版新课标六三制七、八年级数学  
> 项目位置：`D:\tools\math-course`

## 1. 课程如何启动

### 推荐方式：一键启动

1. 打开 `D:\tools\math-course`。
2. 双击 `start-course.bat`。
3. 首次启动会自动安装依赖，需要等待几分钟。
4. 浏览器会自动打开：`http://127.0.0.1:5173`。
5. 学习结束后，关闭标题为“math-course dev server”的命令窗口即可停止课程。

电脑需要预先安装 Node.js 和 npm。若提示“npm was not found”，请先安装 Node.js。

### 手动启动

在 PowerShell 中执行：

```powershell
cd D:\tools\math-course
npm install
npm run dev -- --host 127.0.0.1
```

然后打开 `http://127.0.0.1:5173`。

## 2. 第一次使用应该从哪里开始

根据学习目的选择入口。

### 跟随学校进度

点击顶部“教材章节”，按七上、七下、八上、八下和第1—24章查找对应课程。

这种方式适合：

- 学校刚学完一个章节；
- 考试前按教材章节复习；
- 不确定某个知识点属于哪册教材。

### 系统复习或基础薄弱

从首页模块进入，建议顺序为：

1. 总知识地图；
2. 数与式；
3. 方程与不等式；
4. 图形与几何推理；
5. 坐标与函数；
6. 统计与数据；
7. 综合专题；
8. 测评与复盘。

课程顺序是知识脉络复习顺序，不是教材授课顺序。

### 已经知道薄弱知识点

- 点击“搜索”，输入知识点、题型或易错点，例如“移项变号”“全等判定”“一次函数交点”。
- 点击“知识图谱”，查看它依赖哪些前置知识。
- 若前置节点也薄弱，应先回补前置课程。

## 3. 每节课的正确学习流程

不要从头连续读到尾。建议严格执行以下顺序。

### 第一步：确认知识网络位置

先阅读课首“本节课在知识网络中的位置”：

- 属于哪条知识主线；
- 对应哪一章教材；
- 前置知识是什么；
- 本节只解决哪个核心问题；
- 后续会在哪里使用；
- 容易和什么混淆。

如果前置知识看不懂，先点击对应前置讲次，不要硬学后面的内容。

### 第二步：完成3分钟启动

1. 阅读“今天只做一件事”。
2. 独立完成热身题。
3. 做完后再展开答案。
4. 阅读注意力提醒。

热身题做错时，不需要批评或延长学习时间，先回补它对应的前置知识。

### 第三步：按短时单元学习

推荐节奏：

| 单元 | 时间 | 要求 |
|---|---:|---|
| 课前启动 | 3分钟 | 热身并进入状态 |
| 概念讲解 | 8分钟 | 说清概念从哪里来 |
| 例题讲解 | 10分钟 | 学会题型识别和第一步 |
| 休息 | 3分钟 | 离开屏幕、喝水、活动 |
| 变式训练 | 10分钟 | 独立作答，不照抄例题 |
| 错题复盘 | 5分钟 | 写错因和下次提醒语 |
| 总结 | 2分钟 | 用一句话说出本节核心 |

出现“暂停一下、检查一下、想一想、说一说”时，应先完成提示再继续。

### 第四步：每道题执行5秒刹车

下笔前固定完成：

1. 圈关键词和限制词；
2. 写清题目要求什么；
3. 判断题型并说出第一步；
4. 回忆所属知识主线；
5. 确认后再下笔。

重点不是数满5秒，而是形成“先停、再想、再做”的动作。

### 第五步：使用题型网络

展开例题中的“题型网络”，回答：

- 这类题考什么；
- 第一眼看什么；
- 第一步做什么；
- 容易和什么混淆；
- 条件改变后怎样变式；
- 后续会升级成什么；
- 做错通常属于哪类错因。

看完例题后，必须独立做至少一道变式题。

### 第六步：完成课程并更新状态

课程状态含义：

- 未开始：尚未学习；
- 学习中：正在完成课程；
- 需要复习：基本理解，但还不稳定；
- 容易出错：同类错误反复出现；
- 已掌握：能独立完成变式并通过延迟复习。

不要因为“看懂答案”就标记为已掌握。

## 4. 错题本如何使用

### 记录错题

做错练习时填写：

1. 原题；
2. 原错误答案和错误过程；
3. 主要错因，必要时再选一个次要错因；
4. 一句具体的错误原因；
5. 正确答案或正确方法。

七类错因包括：审题、概念、方法、计算、步骤、检查、迁移错误。

### 错题四问

每道错题订正后回答：

1. 我错在哪里？
2. 我为什么会这样想？
3. 下次看到类似题，第一步做什么？
4. 换一道同类型题，能否独立做对？

### 自动诊断

进入顶部“错题本”，点击“全部自动诊断”。

- 有真实错误证据的知识点会显示为确认断点。
- 只有前置关系但缺少错误证据的节点只是自测候选，不代表已经断裂。
- 应先修复最基础的确认断点，再回到原题。

## 5. D0—D14间隔复习怎么做

| 阶段 | 时间 | 任务 |
|---|---|---|
| D0 | 当天 | 不看答案重做原题，写出错因 |
| D1 | 第二天 | 做同类型基础题 |
| D3 | 第三天 | 做条件或问法变化的变式题 |
| D7 | 一周后 | 混入其他题型，先识别再作答 |
| D14 | 两周后 | 做包含该知识点的综合题 |

进入“错题本”即可看到今日新学、昨日回顾、三日前错题、七日前混合题和十四日前综合题。

只有D14能够独立做对、步骤完整并说清第一步，才算真正掌握。

## 6. 知识图谱怎么用

知识图谱不是用来一次背完的。

推荐用法：

1. 搜索当前不会的知识点；
2. 查看它的前置节点；
3. 判断前置节点是否也薄弱；
4. 找到最基础的薄弱节点；
5. 先学习对应课程，再沿图谱返回当前问题。

每次最多追溯一到两层，避免一次打开过多知识点。

## 7. 学习进度怎么理解

点击顶部“学习进度”可以查看：

- 已开始课程；
- 已掌握课程；
- 各模块掌握比例；
- 本机累计学习时间；
- 每节课完成标准。

进度百分比按“已掌握”计算，不按打开过的课程数量计算。

## 8. 家长如何陪伴

每次陪伴不超过20分钟，只观察一个具体习惯。

不要说：

> 这么简单你怎么又错？

可以说：

> 这次不是新问题，我们今天只处理这一类旧错误。

建议家长只做三件事：

1. 提醒孩子完成5秒刹车；
2. 让孩子自己说出错因和第一步；
3. 孩子自己发现错误时先肯定自查行为。

不要直接讲完整答案，也不要一次纠正多个习惯。

## 9. 推荐的一周安排

| 日期 | 学习任务 |
|---|---|
| 周一 | 新课1节 + D0 |
| 周二 | D1快测 + 新课1节 |
| 周三 | 只处理错题和前置断点 |
| 周四 | 周一内容D3变式 + 新课1节 |
| 周五 | 混合练习或口头讲题 |
| 周末 | 本周课程总结，不集中补大量新课 |
| 下周一 | 完成周一内容D7混合 |
| 两周后 | 完成D14综合检查 |

基础不稳时，每周学习3节新课即可，不要求快速刷完48讲。

## 10. 学习记录保存说明

- 课程状态、错题、复习记录和完成标准保存在当前浏览器的 localStorage。
- 没有账号和云同步。
- 更换浏览器、使用隐私模式或清理浏览器网站数据，可能导致记录丢失。
- 建议固定使用同一台电脑和同一个浏览器配置文件。

### 修改后的验证命令

```powershell
npm run audit:quality
npm run audit:course
npm run audit:formulas
npm run export:samples
npm run lint
npm run build
npm audit
```

所有命令通过后，才能认为修改完成。

### 修改课程的推荐顺序

1. 先核对教材章节边界；
2. 判断是否需要改正文，能用共享层解决时不要重写课程；
3. 保留前置—核心—后续关系；
4. 补题型第一步、易错点和变式；
5. 更新相关结构化数据；
6. 运行全部审计和构建命令；
7. 重新导出代表性样本。

## 12. 常见问题

### 双击脚本后没有打开页面

等待命令窗口出现“Local”地址，再手动打开 `http://127.0.0.1:5173`。

### 5173端口被占用

执行：

```powershell
npm run dev -- --host 127.0.0.1 --port 5174
```

然后打开 `http://127.0.0.1:5174`。

### 公式太长

公式区域支持横向滚动。在手机上不要缩小整个页面，直接左右滑动公式区域。

### 没看到动画

并非每节课都有动画。可以先确认当前课程是否显示“动图演示”模块。

### 错题记录消失

检查是否更换浏览器、使用隐私模式或清理了网站数据。当前版本无法从云端恢复。

## 13. 最重要的使用原则

不要把课程当成需要快速看完的48篇文章。每节课真正完成的标准是：

> 知道它在知识网络中的位置 → 看清题目 → 判断题型 → 说出第一步 → 独立完成 → 分类错因 → 做对变式 → 完成间隔复习。
